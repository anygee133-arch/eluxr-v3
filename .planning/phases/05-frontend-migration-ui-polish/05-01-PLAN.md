**Part D: Update goToPhase() for directional tab transitions**

Find the `goToPhase(n)` function (around line 2385). It currently hides all sections and shows the target with a fade-up transition using the `visible` class and a double-rAF trick.

Update it to:
1. Store the previous phase before updating
2. Determine slide direction (forward = right-to-left, backward = left-to-right)
3. Apply the appropriate slide class instead of (or in addition to) the visibility transition

Replace the function with:

```javascript
function goToPhase(n) {
  const previousPhase = currentPhase;
  currentPhase = n;

  // Update nav items
  document.querySelectorAll('.nav-item').forEach((item, idx) => {
    item.classList.toggle('active', idx === n - 1);
  });

  // Hide all sections, remove old animation classes
  document.querySelectorAll('.page-section').forEach(s => {
    s.classList.remove('active', 'visible', 'slide-in-left', 'slide-in-right');
    s.style.display = 'none';
  });

  // Show target section with directional slide
  const target = document.getElementById(`phase-${n}`);
  if (target) {
    target.style.display = 'block';
    target.classList.add('active');

    // Determine slide direction
    const slideClass = n > previousPhase ? 'slide-in-right' : 'slide-in-left';

    requestAnimationFrame(() => {
      requestAnimationFrame(() => {
        target.classList.add('visible', slideClass);
      });
    });
  }
}
```

**Important:** Preserve any other logic inside the existing goToPhase() function. The function may do more than just show/hide sections (e.g., trigger data loading for specific tabs). Read the existing function carefully and keep any additional behavior. Only modify the show/hide and animation logic.
