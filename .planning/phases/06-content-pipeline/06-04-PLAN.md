  <action>
Update the 02-Theme-Generator workflow. **Keep the existing Auth Validator chain** (Webhook -> Auth Validator -> Auth OK? -> 401 Unauthorized). Replace everything after the Auth OK? true path.

**New flow after Auth OK? (true path):**

1. **"Read ICP" (HTTP Request node):** GET from Supabase `/rest/v1/icps?user_id=eq.{user_id}&select=*`. Use service_role auth. Set `neverError: true`, `alwaysOutputData: true`.

2. **"Read Products" (HTTP Request node):** GET from Supabase `/rest/v1/products?user_id=eq.{user_id}&is_active=eq.true&select=*`. Use service_role auth.

3. **"Prepare Theme Prompt" (Code node):** Build a comprehensive Claude prompt for theme generation. Include:
   - ICP summary, content pillars, pain points, desires
   - Full product list with names, descriptions, categories
   - Month being generated for
   - Instructions for the Netflix model:

   Ask Claude to generate a JSON object:
   ```json
   {
     "show_name": "The [Catchy Name] Series",
     "show_concept": "Brief description of the month's overarching narrative",
     "seasons": [
       {
         "week_number": 1,
         "theme_name": "Season 1: [Name]",
         "inspirational_theme": "Introduction/Discovery theme",
         "season_arc": "intro",
         "theme_description": "What this week covers",
         "hook": "Weekly hook teasing next season (for last post of week)",
         "days": [
           {
             "day_number": 1,
             "product_id": "uuid-of-product",
             "product_name": "Product Name",
             "topic": "Inspirational daily topic tied to product + theme",
             "content_angle": "How to present product through this theme lens"
           },
           ... (7 days)
         ]
       },
       ... (4 seasons)
     ]
   }
   ```

   Product rotation rules:
   - If fewer products than 30 days: products repeat (Claude distributes evenly)
   - If more than 30 products: Claude selects top 30 based on ICP relevance
   - Each day gets exactly 1 product + 1 inspirational topic
   - Products should cluster by theme coherence (similar products in same week when possible)

   Progressive arc rules:
   - Season 1: "intro" -- introduce the brand, products, and value proposition
   - Season 2: "deepen" -- go deeper into product benefits, customer stories, expertise
   - Season 3: "challenge" -- address objections, myths, comparisons
   - Season 4: "celebrate" -- success stories, community, future vision

   Weekly hooks: Last post of each week teases the next season. Daily posts stand alone within their week.

4. **"Generate Themes" (HTTP Request to Claude API):** Call Claude with the prepared prompt. Model: `claude-sonnet-4-20250514`, max_tokens: 8192. Use Claude credential.

5. **"Parse Theme Response" (Code node):** Robust JSON parsing of Claude's response (strip markdown fences, regex extract). Validate structure has show_name, 4 seasons, 7 days per season (28 days minimum -- weeks may have fewer if month is shorter).

6. **"UPSERT Campaign" (HTTP Request node):** POST to `/rest/v1/campaigns?on_conflict=user_id,month` with `Prefer: resolution=merge-duplicates,return=representation`. Body: `{ user_id, month, status: "generating", show_name: parsed.show_name }`.

7. **"Delete Old Themes" (HTTP Request node):** DELETE from `/rest/v1/themes?campaign_id=eq.{campaign_id}` (using campaign_id from UPSERT response). Clears any previous themes for this campaign.

8. **"Prepare Theme Rows" (Code node):** Map the 4 seasons from Claude's response to 4 theme table rows:
   ```javascript
   seasons.map(s => ({
     user_id,
     campaign_id,
     week_number: s.week_number,
     theme_name: s.theme_name,
     theme_description: s.theme_description,
     show_concept: parsed.show_concept,
     hook: s.hook,
     season_arc: s.season_arc,
     inspirational_theme: s.inspirational_theme,
     content_types: JSON.stringify(s.days) // JSONB stores daily assignments
   }))
   ```

9. **"Insert Themes" (HTTP Request node):** POST to `/rest/v1/themes` with the 4 theme rows array. Use service_role auth. Add `Prefer: return=representation`.

10. **"Update Campaign Status" (HTTP Request node):** PATCH `/rest/v1/campaigns?id=eq.{campaign_id}` with `{ status: "active" }`.

11. **"Callback to Orchestrator" (HTTP Request node):** POST to `callback_url` with `{ pipeline_run_id, step: 2, status: "complete" }` and `X-Pipeline-Secret` header. Set `neverError: true`.

12. **"Respond Success" (Respond to Webhook node):** Return `{ success: true, campaign_id, show_name, themes_count: 4 }`.

**Error handling:** If Claude returns unparseable JSON or missing seasons, callback with `status: "error"`. If Supabase operations fail, callback with error.

**n8n JSON patterns:**
- Same node ID conventions, Code node v2 with `$input.item.json`
- All HTTP Request nodes typeVersion 4.2
- Supabase auth: dual headers (Authorization: Bearer SERVICE_ROLE_KEY, apikey: SERVICE_ROLE_KEY)
- `neverError: true` on all Supabase and Claude calls
