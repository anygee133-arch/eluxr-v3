  <action>
Rebuild the 04-Content-Studio workflow for weekly batch content generation. **Keep the existing Auth Validator chain** (Webhook -> Auth Validator -> Auth OK? -> 401). Replace everything after Auth OK? true path.

**Design principle:** This workflow is called once per week (4 times total per campaign). Each execution must complete within 5 minutes. It generates: platform-specific posts + 7 hero images + 3-4 video scripts for 1 week.

**New flow after Auth OK? (true path):**

**Phase A: Load Context**

1. **"Extract Params" (Code node):** Extract from incoming data: user_id, week_number, month, platforms (array), pipeline_run_id, callback_url, business_url. Calculate step number: `step = week_number + 2` (Week 1 = step 3, Week 4 = step 6).

2. **"Read Theme" (HTTP Request):** GET `/rest/v1/themes?user_id=eq.{user_id}&week_number=eq.{week_number}&select=*,campaigns(id,show_name,month)` from Supabase. Use `neverError: true`. This returns the theme row with nested campaign data (PostgREST embedding).

3. **"Read ICP" (HTTP Request):** GET `/rest/v1/icps?user_id=eq.{user_id}&select=icp_summary,content_pillars,recommended_hashtags` from Supabase.

4. **"Read Products" (HTTP Request):** GET `/rest/v1/products?user_id=eq.{user_id}&is_active=eq.true&select=id,name,description,price,image_url` from Supabase.

**Phase B: Generate Posts (1 Claude call per day, batching all platforms)**

5. **"Prepare Daily Content Loop" (Code node):** Parse theme.content_types JSONB to get the 7 daily assignments (day_number, product_id, product_name, topic, content_angle). For each day, prepare a Claude prompt that generates posts for ALL selected platforms in a single call. Mark days 2, 4, 6 (and optionally 7 if week needs 4 video scripts) as "video days" (Claude's discretion on which 3-4 days).

   **Claude prompt per day:**
   Generate social media posts for these platforms: {platforms.join(', ')}

   Context:
   - Show: "{show_name}"
   - Season {week_number}: "{theme_name}" -- {inspirational_theme}
   - Day's Product: {product_name} -- {product_description} (Price: {price})
   - Day's Topic: "{topic}"
   - Content Angle: "{content_angle}"
   - ICP: {icp_summary (truncated to 500 chars)}

   Platform rules (per user's locked decision):
   - LinkedIn: 1300-1800 chars, thought leadership tone, line breaks between paragraphs, 3-5 professional hashtags, product CTA
   - Instagram: 150-300 char caption, 20-30 hashtags in separate first_comment field, emoji allowed, product CTA
   - X: 5-7 tweet thread (each under 280 chars, numbered), punchy and conversational, product CTA in last tweet
   - YouTube: Community post format, story-driven, 200-400 chars, product CTA

   Return JSON:
   ```json
   {
     "posts": {
       "linkedin": { "text": "...", "hashtags": [...], "cta": "..." },
       "instagram": { "text": "...", "hashtags": [...], "first_comment": "...", "cta": "..." },
       "x": { "text": "...", "hashtags": [...], "cta": "..." },
       "youtube": { "text": "...", "hashtags": [...], "cta": "..." }
     },
     "image_prompt": "Product + lifestyle blend image prompt for the day (max 500 chars)"
   }
   ```
   Only include platforms listed. Product CTA ties back to featured product.

   Also generate a hero image prompt: product as hero in aspirational lifestyle scene, matching the season theme, 1:1 aspect ratio, photorealistic, no text overlays.

6. **"Generate Day Content" (Loop Over Items + HTTP Request to Claude):** Loop through 7 days. For each day, call Claude API (claude-sonnet-4-20250514, max_tokens 4096) with the day's prompt. Use Claude credential. Add a 2-second Wait node between iterations to avoid rate limits (per pitfall MOD-6).

7. **"Parse Day Content" (Code node, inside loop):** Robust JSON parse each Claude response. Extract platform posts + image prompt. Map to content_items rows:
   For each platform in response:
   ```javascript
   {
     user_id,
     campaign_id,
     theme_id,
     title: `${show_name} S${week_number} Day ${day_number} - ${platform}`,
     content: post.text,
     content_type: "text", // All posts are text type
     platform,
     scheduled_date: calculateDate(month, week_number, day_number), // e.g., 2026-03-01 for W1D1
     status: "pending_review",
     image_prompt: dayContent.image_prompt, // Same for all platforms that day
     product_id: daily_assignment.product_id,
     hashtags: post.hashtags,
     first_comment: post.first_comment || null
   }
   ```

**Phase C: Generate Video Scripts (3-4 per week)**

8. **"Research Trending Audio" (HTTP Request to Perplexity):** One Perplexity call per week to find trending audio. Prompt: "What are the top 10 trending sounds/songs on Instagram Reels and YouTube Shorts right now in {month}? For each, provide: song name, artist, and which platform it's trending on. Focus on sounds that work well for {industry} content." Model: sonar-pro. Credential: 3rYRY1C2K9o0DDXI.

9. **"Generate Video Scripts" (Code node -> Loop + Claude):** For the 3-4 video days this week, generate detailed production-ready video scripts via Claude. Each script should be a JSON object:
   ```json
   {
     "title": "Video title",
     "duration": "30 seconds",
     "platforms": ["Instagram Reels", "YouTube Shorts"],
     "trending_audio": [
       { "name": "...", "artist": "...", "platform": "Instagram" },
       { "name": "...", "artist": "...", "platform": "YouTube" }
     ],
     "sections": {
       "hook": { "timing": "0:00-0:03", "script": "...", "camera": "...", "b_roll": "...", "on_screen_text": "..." },
       "setup": { "timing": "0:03-0:10", "script": "...", "camera": "...", "b_roll": "...", "on_screen_text": "..." },
       "value": { "timing": "0:10-0:25", "script": "...", "camera": "...", "b_roll": "...", "on_screen_text": "...", "key_points": [...] },
       "cta": { "timing": "0:25-0:30", "script": "...", "camera": "...", "on_screen_text": "...", "action": "..." }
     },
     "music_mood": "...",
     "thumbnail_idea": "...",
     "production_notes": "...",
     "product_url": "...",
     "product_image": "..."
   }
   ```

   Add the product's image_url and source_url to each script so a videographer has all references.

   Store video scripts in the corresponding content_items by adding the `video_script` JSONB field. Also create a separate content_item with `content_type: "video"` for each video day (platform: based on instagram/youtube presence in selected platforms).

**Phase D: Batch Insert Content Items**

10. **"Batch Insert Content" (HTTP Request):** POST all content_items for this week to `/rest/v1/content_items` as a JSON array. Batch insert (not individual). Use service_role auth. Add `Prefer: return=representation` to get back IDs.

**Phase E: Generate Hero Images (7 per week)**

11. **"Generate Images Loop" (Loop Over Items):** For each of the 7 daily image prompts:
   a. **"Create KIE Task" (HTTP Request):** POST to `https://api.kieai.com/v1/tasks` with:
      ```json
      { "model": "nano-banana-pro", "input": { "prompt": "...", "aspect_ratio": "1:1", "resolution": "1K", "output_format": "png" } }
      ```
      Use KIE credential.
   b. **"Wait Initial" (Wait node):** 10 seconds.
   c. **"Poll KIE Result" (HTTP Request):** GET `https://api.kieai.com/v1/tasks/{task_id}`. Use KIE credential.
   d. **"Check Result" (Code node):** Check if `output.status === 'succeeded'`. Extract `output.works[0].resource.url` as image_url.
   e. **"IF Complete" (IF node):** If succeeded -> update content_items. If not -> "Wait Retry" (5s) -> Poll again (max 12 attempts).
   f. **"Update Image URL" (HTTP Request):** PATCH `/rest/v1/content_items?campaign_id=eq.{campaign_id}&scheduled_date=eq.{date}&image_url=is.null` with `{ image_url: kieImageUrl }`. Updates all platform posts for that day with the shared hero image.

   Use the same Wait+IF polling pattern from existing workflow 11 (already proven working). Add 2-second gap between image generation tasks to avoid KIE rate limits.

**Phase F: Callback**

12. **"Callback to Orchestrator" (HTTP Request):** POST to `callback_url` with `{ pipeline_run_id, step: week_number + 2, status: "complete" }`. Set `neverError: true`.

13. **"Respond Success" (Respond to Webhook):** Return `{ success: true, week_number, content_count, images_generated }`.

**Timing budget for 5-minute limit:**
- Load context: ~3s
- 7 Claude calls (posts) with 2s gaps: ~35s
- 1 Perplexity call (trending audio): ~5s
- 3-4 Claude calls (video scripts) with 2s gaps: ~20s
- Batch insert: ~2s
- 7 KIE image generations with ~15s each: ~105s
- DB updates: ~5s
- Total: ~175s (~3 minutes) -- well within 5-minute limit

**Error handling:** If any Claude call fails to parse, skip that day's content and note in metadata. If KIE image generation fails for a day, skip and leave image_url null. Always call callback (even on partial failures) with status "complete" (partial content is better than no content).

**Date calculation:** Use the month (YYYY-MM) + week_number + day_number to calculate scheduled_date. Week 1 Day 1 = 1st of month. Week 1 Day 7 = 7th. Week 2 Day 1 = 8th. Etc. Handle months with fewer than 28 days (February) and months with 30/31 days gracefully.
