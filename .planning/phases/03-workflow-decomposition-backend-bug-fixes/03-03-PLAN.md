<objective>
Build and deploy sub-workflows 06-10 (Approval List, Approval Action, Clear Pending, Calendar Sync, Chat), replacing all Google Sheets nodes with Supabase queries. Collapse the find+update Google Sheets pairs into single Supabase PATCH operations.

Purpose: These 5 workflows handle the approval/review pipeline and supporting features (calendar sync, chat). The approval workflows have the highest density of Google Sheets nodes (11 of 16 total), making them the bulk of the INFRA-03 migration. Collapsing find+update pairs into single PATCH calls simplifies the workflows and eliminates race conditions.

Output: 5 independently deployable sub-workflows with all approval-related Google Sheets nodes replaced by Supabase operations. Google Calendar node preserved as-is (deferred to Phase 8).
</objective>

<task type="auto">
  <name>Task 2: Build 07-Approval-Action sub-workflow</name>
  <files>n8n-workflow-remote</files>
  <action>
This workflow has the most Google Sheets nodes to replace (8 nodes collapse to 3). Extract the approval action endpoint.

**Nodes to extract:**
- Approval Action Webhook (webhook: /eluxr-approval-action)
- Auth Validate (eluxr-approval-action) + Auth OK? + 401 Unauthorized
- Is Approve? (IF node)
- Find Content Row + Update to Approved -> **COLLAPSE to single Supabase PATCH**
- Is Reject? (IF node)
- Find Row for Reject + Update to Rejected -> **COLLAPSE to single Supabase PATCH**
- Find Row for Edit + Update Content -> **COLLAPSE to single Supabase PATCH**
- Respond Success (Respond to Webhook)

**Google Sheets collapse pattern:**

In the monolith, each action requires two Google Sheets operations (Find Row, then Update Row). With Supabase, a single PATCH with a filter replaces both:

**Approve action:**
- Method: PATCH
- URL: `https://llpnwaoxisfwptxvdfed.supabase.co/rest/v1/content_items?id=eq.{{ $json.body.content_id }}&user_id=eq.{{ $json.user_id }}`
- Body: `{ "status": "approved", "updated_at": "{{ $now.toISO() }}" }`
- Headers: Prefer: return=representation

**Reject action:**
- Method: PATCH
- URL: `https://llpnwaoxisfwptxvdfed.supabase.co/rest/v1/content_items?id=eq.{{ $json.body.content_id }}&user_id=eq.{{ $json.user_id }}`
- Body: `{ "status": "rejected", "feedback": "{{ $json.body.feedback }}", "updated_at": "{{ $now.toISO() }}" }`

**Edit action:**
- Method: PATCH
- URL: `https://llpnwaoxisfwptxvdfed.supabase.co/rest/v1/content_items?id=eq.{{ $json.body.content_id }}&user_id=eq.{{ $json.user_id }}`
- Body: `{ "content": "{{ $json.body.content }}", "updated_at": "{{ $now.toISO() }}" }`

**IMPORTANT:** Include `user_id=eq.{{ $json.user_id }}` in every PATCH URL filter for tenant isolation -- this prevents a user from modifying another user's content even if they know the content_id.

**Workflow structure:**
```
Webhook -> Auth -> IF authenticated
  -> IF action = "approve" -> Supabase PATCH (status: approved) -> Respond
  -> IF action = "reject" -> Supabase PATCH (status: rejected) -> Respond
  -> ELSE (edit) -> Supabase PATCH (content: updated) -> Respond
```

Or use a Switch node to route by action type.

**Deploy** via `n8n_create_workflow` with name "07-Approval-Action".

Save to `workflows/07-approval-action.json`.
