**ROOT CAUSE FIX — All products showing same item in Step 4:**
- `Read Products` returns 31 items (one per product row). `Read Weekly Topics` was connected downstream, so n8n ran it **31 times** — producing 31 × 7 = 217 topic items.
- `Prepare Daily Content Loop` used `.first()` which got ONE topic object (not the array). `Array.isArray(singleObject)` returned false, so `weeklyTopics = []`. Code fell through to `theme.content_types` fallback which had a single product for all 7 days.

## Key Debugging Insight

The n8n HTTP Request node splits JSON array responses into individual items. When nodes are chained sequentially (Read ICP → Read Products → Read Weekly Topics → Prepare Daily Content Loop), each downstream node runs once per upstream item. With 31 products, Read Weekly Topics executed 31 times producing 217 items. Using `.first()` or `.item` on such nodes returns a single item, not the full dataset. The fix is `.all()` with deduplication.
